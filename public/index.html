Privacy Policy

Data We Collect

When you register, we collect your name, email address, and a hashed version of your password. We never store your password in plain text.

API Keys

Your API key is shown once at registration. We store only a SHA-256 hash of it. We cannot recover or view your key after creation.

Usage Data

We log API call metadata (endpoint, status code, response time, timestamp) to provide usage tracking and enforce monthly quotas. We do not store the content of your screenshots or PDFs.

Payment Information

All payment processing is handled by Stripe. We never see, store, or have access to your credit card details. Stripe is PCI DSS Level 1 certified.

Data Security

Passwords are hashed with bcrypt (10 salt rounds). API keys are hashed with SHA-256. All data is transmitted over HTTPS in production. Security headers are enforced via Helmet.

No Sharing

We do not sell, share, or distribute your personal data to any third party. Your data is used solely to operate the Scrave service.

Data Deletion

You may request deletion of your account and all associated data by contacting us. Upon deletion, all user data, API keys, and usage logs will be permanently removed.

Contact

For any privacy-related questions, reach out via our Discord community or Twitter.

About Scrave

What is Scrave?

Scrave is a developer-first API that converts any public webpage into pixel-perfect screenshots (PNG/JPEG) or PDF documents. One API call, headless Chromium rendering, zero infrastructure to manage.

Why does it exist?

Developers need web captures for reporting, social cards, invoices, visual testing, and archiving. But self-hosting headless browsers is fragile and expensive, and enterprise APIs charge $100+/month.

Scrave offers a simple, affordable middle ground — a generous free tier, transparent pricing, and an API you can set up in 30 seconds from the terminal.

How it works

1 Register and get your sf_live_ API key
2 Send a POST request with a URL to /api/v1/screenshot or /api/v1/pdf
3 Get back the rendered image or PDF — that's it
